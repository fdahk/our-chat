# 03 · 开发流程

## 1. 首次环境搭建

```bash
# 装工具(只需一次,全局)
brew install tuist swiftlint swiftformat

# 可选:用 mise 锁定工具版本(团队协作推荐)
brew install mise
cd mobile-swift && mise install
```

## 2. 拉代码后的初始化

```bash
cd mobile-swift
tuist install      # 拉 SPM 依赖,落在 Tuist/Dependencies/(被 gitignore)
tuist generate     # 生成 OurChat.xcworkspace
open OurChat.xcworkspace
```

`OurChat.xcworkspace` **不入库**。每次切分支后如果发现 Project.swift 或 Tuist/Package.swift 有改动,重跑 `tuist generate`。

## 3. 日常开发命令

| 干嘛 | 命令 |
|------|------|
| 加依赖 | 编辑 `Tuist/Package.swift` → `tuist install` → `tuist generate` |
| 改工程结构 | 编辑 `Project.swift` → `tuist generate` |
| 跑模拟器 | Xcode 里 ⌘+R |
| 跑单元测试(Xcode) | ⌘+U |
| 跑单元测试(命令行) | `tuist test` |
| 单跑某个测试 | `tuist test --filter <YourFeature>Tests` |
| 检查格式 | `swiftformat --lint Sources Tests` |
| 自动格式化 | `swiftformat Sources Tests` |
| 检查 lint | `swiftlint lint --quiet` |
| 清理 Derived | `rm -rf Derived && tuist clean` |

## 4. Git 工作流

本项目位于 `/Users/mac/our-chat/` 这个**已有 git 仓库**的子目录。提交时:

```bash
cd /Users/mac/our-chat
git add mobile-swift/
git commit -m "feat：new record"
```

不要在 mobile-swift/ 下单独 `git init`。

**不要入库的文件**(已写入 `.gitignore`):
- `Derived/` — Tuist 缓存
- `OurChat.xcodeproj/` / `OurChat.xcworkspace/` — Tuist 生成产物
- `.build/` — SPM build cache
- `Package.resolved` — 依赖版本锁文件(本项目用 Tuist 管理,resolved 文件不稳定)
- `.swiftpm/` — Xcode SPM cache
- `xcuserdata/` — 个人 Xcode 状态

**应该入库**:
- `Project.swift` / `Tuist/Package.swift` — 工程清单
- `Sources/` / `Tests/` / `Resources/` — 业务代码
- `.swiftformat` / `.swiftlint.yml` / `.gitignore` / `.mise.toml` — 配置
- `docs/` / `README.md`

## 5. 提交前自检清单

```bash
# 1. 格式
swiftformat Sources Tests
# 2. Lint
swiftlint lint --quiet
# 3. 测试全绿
tuist test
# 4. 看看改了什么
git diff --stat
```

四步通过再 commit。**测试失败时不要 force push 或 skip,直接修**。

## 6. SwiftUI Preview 工作流

Sources/Features/.../XxxView.swift 文件末尾的 `#Preview {}` 是 SwiftUI 内置预览。

在 Xcode 编辑器右侧 Canvas 自动渲染,改代码即时刷新。Preview 使用 `chatClient.previewValue` 而非 `liveValue`,所以**离线、零网络、零数据库**也能渲染。

## 7. 调试技巧

### TCA `_printChanges()`

在 Store 创建时挂 `_printChanges()`,Debug build 控制台会打印每个 action 的 state diff,等价 Redux DevTools 的 action log。例:

```swift
@MainActor static let store = Store(initialState: AppFeature.State()) {
    AppFeature()
        ._printChanges()
}
```

### Swift Concurrency 死锁排查

如果点击不响应、UI 卡死,优先怀疑:
- 主线程 sync 等待 actor(用 await 替换)
- `@MainActor` 嵌套调用循环
- TaskGroup 没设 cancellation

用 Xcode 的 Debug > View Process > Suspend All Threads 看主线程栈。

### Network Inspector

Xcode 26 内置 Network Inspector(WWDC 2024 引入),勾上 `URLSession` 任务会自动捕获请求/响应。不需要 Charles 类的第三方工具。

## 8. 测试覆盖率

```bash
tuist test --result-bundle-path ./TestResults.xcresult
# 用 xcrun xccov 看覆盖率
xcrun xccov view --report ./TestResults.xcresult
```

目标(本项目):
- Feature (Reducer): **覆盖 ≥ 90%**(TCA 设计就是为高覆盖)
- Service (Client): **覆盖 ≥ 70%**
- View: **不强制单测**,用 swift-snapshot-testing 做快照覆盖

## 9. 真机调试

第一次跑真机需要:
1. Xcode → Settings → Accounts 登录 Apple ID(免费即可)
2. 选 Target → Signing & Capabilities → Team 选自己
3. iPhone 设置 → 隐私与安全性 → 开发者模式 → 打开 → 重启
4. iPhone 用 USB-C 接电脑,信任此电脑
5. Xcode 顶部 Run 目标选你的真机 → ⌘+R

免费证书 7 天后失效,App 自动失效。要持久部署需要 99 USD/年 Apple Developer Program。
