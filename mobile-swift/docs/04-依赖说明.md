# 04 · 依赖说明

每个 SPM 依赖的作用、版本锚定原因、对应的前端类比。新增依赖前先读本文,避免引入"过时栈"或重复职责。

## 当前依赖清单(`Tuist/Package.swift`)

### 1. swift-composable-architecture(TCA)· `from: 1.17.0`

- **作用**:状态管理 + 架构骨架
- **前端类比**:Redux Toolkit + RTK Query + Immer + react-redux 一体化
- **版本锚定原因**:1.17 引入 `@Reducer` + `@ObservableState` 宏稳定版,锁主版本 1.x
- **替代品**:
  - Apple 官方 MV pattern(@Observable + Environment)— 小项目够用,大项目失控
  - Bloc(Flutter 移植)— Swift 版本不活跃
  - Spin / ReSwift — 社区维护放缓
- **触发引入条件**:已是核心架构,Day 1 引入

### 2. swift-dependencies · `from: 1.6.0`

- **作用**:依赖注入框架,`@Dependency` 注入 + `@DependencyClient` 自动生成 mock
- **前端类比**:inversify(但更轻量)+ MSW 的 handler 思路
- **版本锚定原因**:1.6+ 才有 `@DependencyClient` 宏稳定版
- **替代品**:
  - Swinject — OOP 风格,过时
  - Factory — 适合非 TCA 项目
  - 手写 protocol + struct — 重复代码爆炸
- **触发引入条件**:已与 TCA 强绑定,必装

### 3. DependenciesMacros · 随 swift-dependencies 一起

`@DependencyClient` 宏的实现。`Sources/Services/` 下的每个 `@DependencyClient` struct 都依赖此包。

### 4. swift-snapshot-testing · `from: 1.18.0`

- **作用**:UI 快照测试 + JSON 序列化对比测试
- **前端类比**:Jest Snapshot + Storybook 截屏对比
- **版本锚定原因**:1.18 兼容 Swift Testing 注解
- **替代品**:
  - 手写 XCTAssertEqual 字符串对比 — 痛苦
  - GitHub-actions 的 Pixel-diff workflow — CI 慢
- **触发引入条件**:Day 1 引入(防止 UI regression)

### 5. Kingfisher · `from: 8.0.0`

- **作用**:图片远程加载 + 内存/磁盘缓存 + WebP/HEIC 解码
- **前端类比**:next/image + sharp + react-image
- **版本锚定原因**:8.x 是 Swift 6 / strict concurrency 第一个完整支持版本
- **替代品**:
  - SwiftUI 内置 `AsyncImage` — 简单场景够用,缺缓存策略和高级用法
  - Nuke — 性能更好但 API 更底层
  - SDWebImage — OC 时代标准答案,Swift 时代逐渐淘汰
- **触发引入条件**:首次需要展示用户头像 / 远程图片时

### 6. Mockable · `from: 0.3.0`

- **作用**:基于 Swift macro 自动给 protocol 生成 mock
- **前端类比**:vitest-mock-extended + ts-mockito
- **版本锚定原因**:0.x 处于早期,锁次版本
- **替代品**:
  - 手写 mock struct — 啰嗦
  - Cuckoo — 已不维护
  - SwiftMock — 启动器 API
- **触发引入条件**:protocol-based 依赖增多时(本项目 Day 1 引入备用)

### 7. GRDB.swift · `from: 7.0.0`

- **作用**:SQLite 现代 Swift 封装,支持 Codable / ValueObservation / DatabasePool 多读单写
- **前端类比**:Prisma + better-sqlite3
- **版本锚定原因**:7.0 是 Swift 6 strict concurrency 第一个完整支持版本
- **替代品**:
  - SwiftData — Apple 官方,iOS 17+,但跨表查询和性能调优能力弱
  - Core Data — 老牌,API 仍然 OC 风格
  - Realm — 不推荐(Atlas Device Sync EOL)
  - FMDB — OC 时代封装,过时
- **触发引入条件**:本地缓存会话/消息历史时(Day 1 引入备用)

## 当前**未引入**但可能会加的

### A. swift-openapi-generator + swift-openapi-urlsession

- **作用**:从 OpenAPI 3.x spec 自动生成 type-safe Swift client
- **前端类比**:openapi-typescript + openapi-fetch
- **触发引入条件**:后端固定 OpenAPI spec 后

### B. swift-async-algorithms

- **作用**:`AsyncStream` 操作符(debounce / throttle / combineLatest)
- **前端类比**:RxJS Subject + operators
- **触发引入条件**:输入框防抖、实时消息流处理变复杂时

### C. swift-collections

- **作用**:OrderedSet / Heap / Deque
- **前端类比**:Immutable.js / immer 的 OrderedMap
- **触发引入条件**:有非平凡数据结构需求时

### D. MLX Swift / CoreML

- **作用**:端侧 AI 推理
- **触发引入条件**:做总结、翻译、智能回复等 AI feature 时

## 全局原则

1. **每个依赖必须解决一个具体问题**——不要因为"别人都用就加"
2. **优先标准库 + Apple 框架**——例如优先 `AsyncImage` 再到 Kingfisher
3. **版本锚 `from: x.y.z`**——主版本不动,次版本可升,patch 自动跟
4. **禁止 Pin to Specific Version**——除非有已知不兼容 PR
5. **依赖增加要在 PR 描述说明 why**——避免"看起来很酷"型引入

## 加新依赖的流程

```bash
# 1. 编辑 Tuist/Package.swift,加一行
.package(url: "https://github.com/xxx/yyy", from: "1.0.0"),

# 2. 把 product 名加到 Project.swift 的 dependencies 数组
.external(name: "Yyy"),

# 3. 拉依赖
tuist install

# 4. 重新生成工程
tuist generate

# 5. 试用一下,跑测试,跑 lint
tuist test
swiftformat Sources && swiftlint --quiet

# 6. 提交时把变化解释清楚
# git diff 应该只包含 Tuist/Package.swift 和 Project.swift,不应有 *.xcodeproj 改动
```
