# 05 · 常见问题排查

按 "现象 → 原因 → 修复" 格式,覆盖搭建到首次跑通最容易卡住的地方。

## 1. `tuist install` 失败

### 1.1 现象:`error: unable to resolve package graph`

```
error: unable to resolve package graph: Cannot find package 'XXX'
```

**原因**:`Tuist/Package.swift` 里包名拼错,或者远程 URL 404。

**修复**:
```bash
# 删 SPM 缓存重试
rm -rf .build .swiftpm Tuist/Dependencies
tuist install
```

### 1.2 现象:`swift package resolve` 卡住不动

**原因**:网络问题,GitHub 拉不动。

**修复**:配置 Git 的 https proxy 或 SSH 替代:
```bash
git config --global url."git@github.com:".insteadOf "https://github.com/"
```

## 2. `tuist generate` 失败

### 2.1 现象:`error: Multiple products were found with the same name`

**原因**:`Tuist/Package.swift` 里某个包暴露了重名 product,或者两个依赖的传递依赖冲突。

**修复**:在 `Tuist/Package.swift` 的 `packageSettings.productTypes` 里显式指定 framework/staticFramework:
```swift
productTypes: [
    "Kingfisher": .framework,
]
```

### 2.2 现象:`error: missing 'Resources' folder`

**原因**:`Project.swift` 里写了 `resources: ["Resources/**"]` 但目录为空。

**修复**:
```bash
mkdir -p Resources/Assets.xcassets/AppIcon.appiconset
mkdir -p Resources/Assets.xcassets/AccentColor.colorset
# 然后填充 Contents.json(本项目已有模板)
```

### 2.3 现象:`No such module 'ComposableArchitecture'`

**原因**:`tuist install` 没跑,或者 `tuist generate` 没重新执行。

**修复**:
```bash
tuist install && tuist generate
```

## 3. 模拟器问题

### 3.1 现象:Cmd+R 后模拟器一直黑屏

**原因**:Simulator 启动卡住,或与 host 时间不同步。

**修复**:
```bash
# 强制重启模拟器服务
xcrun simctl shutdown all
killall Simulator
open -a Simulator
```

### 3.2 现象:`Could not find any iOS simulators`

**原因**:Xcode 26 装好后没下 iOS 18 模拟器 runtime。

**修复**:
```bash
# 在 Xcode 里:Settings → Components → 下载 iOS 18 simulator runtime
# 或命令行:
xcodebuild -downloadPlatform iOS
```

## 4. 编译问题

### 4.1 现象:大量 `Capture of 'self' with non-sendable type ... in a @Sendable closure`

**原因**:Swift 6 严格并发模式下,跨 actor 边界的闭包必须捕获 Sendable 值。

**修复**:三选一:
```swift
// 方案 A: 把要捕获的 struct 标记 Sendable
struct Foo: Sendable { ... }

// 方案 B: 显式拷贝局部变量
let snapshot = self.foo
return .run { send in
    await send(.done(snapshot))
}

// 方案 C: 整个类型迁移到 actor 隔离
@MainActor
final class Foo { ... }
```

详细对照 [docs/ios原生开发/21-Swift 并发 2 Sendable 与 Strict Checking.md](/Users/mac/myReact/docs/ios原生开发/21-Swift%20并发%202%20Sendable%20与%20Strict%20Checking.md)。

### 4.2 现象:`Main actor-isolated property '_X' can not be referenced from a Sendable closure`

**原因**:`@DependencyClient` 自动生成的 endpoint 要求 `@Sendable` 闭包,但你在闭包里访问了非 Sendable 状态。

**修复**:重构服务实现把状态封装进 actor 或 immutable struct,不要让 endpoint 闭包捕获外部可变状态。

### 4.3 现象:`Cannot find type 'IdentifiedArray' in scope`

**原因**:`IdentifiedArray` 是 TCA 内部依赖 `swift-identified-collections`,需要在文件顶部 `import ComposableArchitecture`(它会传递导出)。

**修复**:加 `import ComposableArchitecture`。

## 5. TCA 相关问题

### 5.1 现象:`TestStore` 测试报 `unhandled actions remain`

**原因**:Effect 发出了一个 action,但你没在测试里 `await store.receive(...)`。

**修复**:
```swift
await store.send(.onAppear) {
    $0.isLoading = true
}
// 必须显式接收 Effect 发回的 action
await store.receive(\.conversationsResponse.success) {
    $0.isLoading = false
    $0.conversations = [...]
}
```

### 5.2 现象:Preview 报 `Failed to render: unimplemented dependency`

**原因**:Feature 用了 `@Dependency(\.foo)` 但 Preview 没有 `previewValue`。

**修复**:在 `extension FooClient: DependencyKey` 里补 `static let previewValue = ...`。

### 5.3 现象:Reducer 改了 state 但 View 不刷新

**原因**:
- 用了 `Store` 而非 `@Bindable var store`(SwiftUI 17+ Observable 风格)
- State 没标 `@ObservableState`
- View 用了 `WithViewStore`(旧 API)而非新的 store 直读

**修复**:
```swift
struct XxxView: View {
    @Bindable var store: StoreOf<XxxFeature>  // 不是 let

    var body: some View {
        Text(store.title)  // 直接读,Observable 框架自动追踪
    }
}
```

## 6. Git 问题

### 6.1 现象:`git add .` 误把 `.xcodeproj/` 也加进去

**原因**:可能 `.gitignore` 没生效(被先 add 后才写)。

**修复**:
```bash
git rm -r --cached "*.xcodeproj"
git rm -r --cached Derived
git commit -m "feat：new record"
```

### 6.2 现象:同事 pull 下来跑不起来

**原因**:大概率是 SPM 解析的版本不一致(我们故意不入库 `Package.resolved`,因为 Tuist 在管理)。

**修复**:
```bash
cd mobile-swift
rm -rf .build .swiftpm Tuist/Dependencies
tuist install
tuist generate
```

## 7. SwiftLint / SwiftFormat 问题

### 7.1 现象:`swiftlint: command not found`

```bash
brew install swiftlint
```

### 7.2 现象:CI 上 swiftformat 报红但本地没问题

**原因**:版本不一致。

**修复**:用 mise 锁版本:
```bash
brew install mise
mise install
mise exec -- swiftformat --lint Sources Tests
```

## 8. 真机部署

### 8.1 现象:`No signing certificate "iOS Development" found`

**原因**:没在 Xcode 里登 Apple ID,或没勾 "Automatically manage signing"。

**修复**:Xcode → Settings → Accounts → 登 Apple ID → Target → Signing & Capabilities → 勾 Automatically manage signing → 选 Team。

### 8.2 现象:真机装上 App 启动闪退,Console 显示 `Untrusted Developer`

**原因**:iOS 没信任你的开发者证书。

**修复**:iPhone → 设置 → 通用 → VPN 与设备管理 → 开发者 App → 信任。

## 9. 常用 reset 套路

各种问题排查不出来,用这个核武器:

```bash
# 1. 关掉所有 Xcode 实例
killall Xcode

# 2. 清 Tuist + SPM
cd mobile-swift
rm -rf Derived .build .swiftpm Tuist/Dependencies *.xcodeproj *.xcworkspace

# 3. 清 Xcode 全局 Derived Data
rm -rf ~/Library/Developer/Xcode/DerivedData/*

# 4. 重新拉 + 生成
tuist install
tuist generate
open OurChat.xcworkspace
```

如果还是不行,**不要继续 try**,把现象 + 完整命令日志贴 GitHub Issue。
