# 01 · 本地开发环境搭建

> **目标**:新人 clone 仓库后,**5 分钟跑起本地 server**

## 1. 前置依赖

| 软件 | 最低版本 | 推荐 | macOS 装法 |
|---|---|---|---|
| Node.js | 22 | 22.x LTS | `brew install node@22` 或 nvm |
| pnpm | 10 | 10.x | `npm i -g pnpm@10` |
| Docker | 25 | Docker Desktop / Colima | `brew install --cask docker` |
| Docker Compose 插件 | v2 | 同上(自带) | — |
| OpenSSL | 1.1+ | 系统自带 | macOS / Linux 默认有 |

**验证**:

```bash
node -v        # v22.x
pnpm -v        # 10.x
docker --version
docker compose version
openssl version
```

## 2. 三种启动方式(挑一种,首次推荐 2A)

### 2A · 完全容器化(零依赖,只需 Docker)

适合从未装过 Node / 担心污染本机环境的工程师:

```bash
git clone <repo>
cd server
cp .env.example .env
# 编辑 .env,至少填 JWT_SECRET=$(openssl rand -hex 32)

# 生成 OAuth 私钥到 keys/(会被 server 容器只读挂载)
mkdir -p keys && openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 \
  -out keys/oauth-private-dev.pem && chmod 600 keys/oauth-private-dev.pem

# 起 PG + server,bind mount + tsx watch 热重载
pnpm docker:dev
# 等价于:docker compose -f docker-compose.yml -f docker-compose.dev.yml up
```

改 `src/` 任何文件 → server 容器 tsx watch 自动重启。
访问 <http://localhost:3007/health> 应该 `{"status":"ok"}`。

### 2B · 半容器化(PG 容器,Node 本机)

适合本机已装好 Node + pnpm + 喜欢用 IDE debugger 的工程师:

```bash
git clone <repo>
cd server
pnpm setup     # = bash scripts/setup-dev.sh,~1 分钟
pnpm dev       # 启动 server(本机进程),监听 :3007
```

`pnpm setup` 做的事(每步幂等,可重复执行):

1. **检查依赖**(node / pnpm / docker / openssl)
2. **创建 `.env`**(从 `.env.example` 复制,首次)
3. **生成 OAuth dev RSA 私钥**(`keys/oauth-private-dev.pem`,600 权限,已 .gitignore)
4. **`pnpm install`**(装依赖)
5. **`docker compose up -d postgres`**(起 PG 16-alpine,带 healthcheck + 持久化卷)
6. **等 PG 就绪**(轮询 `pg_isready`)
7. **`pnpm db:generate`**(生成 Prisma Client)
8. **`pnpm db:migrate:deploy`**(应用 `prisma/migrations/0_initial`)

完成后控制台会列出下一步命令。

### 2C · 类生产模式(本地跑 runner image,验证镜像可用)

```bash
cp .env.example .env
# 编辑 .env,至少填 JWT_SECRET
mkdir -p keys && openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 \
  -out keys/oauth-private-dev.pem
pnpm docker:build      # build runner stage 镜像
pnpm docker:up         # 起 server + PG(都是容器)
```

这个模式跟生产部署用同一份镜像,改代码不会自动热重载(需要 rebuild)。**适合验证 Dockerfile 没问题 / 跑集成验收**。

## 3. 分步启动(若不想用脚本)

```bash
# (1) 装依赖
pnpm install

# (2) 准备 .env
cp .env.example .env
# 必填:DATABASE_URL / JWT_SECRET / OAUTH_ISSUER_BASE_URL

# (3) 生成 OAuth 私钥
mkdir -p keys
openssl genpkey -algorithm RSA -pkeyopt rsa_keygen_bits:2048 -out keys/oauth-private-dev.pem
chmod 600 keys/oauth-private-dev.pem

# (4) 起 PostgreSQL
docker compose up -d postgres
# 等待 PG 就绪
until docker compose exec -T postgres pg_isready -U postgres -d our_chat; do sleep 1; done

# (5) 生成 Prisma Client + 应用 Migration
pnpm db:generate
pnpm db:migrate:deploy

# (6) 启动
pnpm dev
```

## 5. 常用日常命令

```bash
# 应用
pnpm dev                本机启动 server(tsx watch 热重载)
pnpm test               跑所有测试
pnpm test:watch         监听模式
pnpm typecheck          TypeScript 严格检查

# Docker(完整 stack:server + PG)
pnpm docker:build       build server runner image
pnpm docker:up          起 server + PG(类生产模式)
pnpm docker:dev         起 server + PG(dev override:bind mount + 热重载)
pnpm docker:pg          只起 PG(供 2B 本机 Node 模式用)
pnpm docker:down        停所有(保留数据卷)
pnpm docker:logs        看所有容器日志
pnpm docker:logs:server 仅看 server 日志
pnpm docker:logs:pg     仅看 PG 日志
pnpm clean:db           停所有 + 删数据卷(下次 docker up 会重建表)

# 数据库
pnpm db:migrate:dev     开发新 schema 变更
pnpm db:migrate:deploy  生产/CI 应用 pending migration
pnpm db:migrate:status  看哪些 migration 已应用
pnpm db:studio          Prisma Studio Web UI(浏览/编辑数据)
```

## 7. 常见问题(FAQ)

### Q1:`pnpm setup` 报"缺 docker"

装 Docker Desktop:<https://www.docker.com/products/docker-desktop>。macOS 用户也可考虑 Colima(更轻):`brew install colima docker docker-compose && colima start`。

### Q2:`pg_isready` 长时间不返回,setup 超时

- 看 `docker compose logs postgres`,通常是端口 5432 被占用(本机已经在跑 PG)
- 改 `docker-compose.yml` 的 `ports: ["5433:5432"]`,然后 `DATABASE_URL=postgresql://postgres:postgres@localhost:5433/our_chat`

### Q3:`pnpm dev` 报 "P3018: 找不到 Migration"

`prisma/migrations/` 目录被改坏了。验证:

```bash
ls prisma/migrations/0_initial/migration.sql
```

正常应该 ~330 行。如果丢失了,从 git 恢复:`git checkout prisma/migrations/`。

### Q4:报 "Prisma Client not generated"

```bash
pnpm db:generate
```

### Q5:报 "EADDRINUSE 3007"

```bash
# 看占用端口的进程
lsof -i :3007
# 或者改 env
PORT=3008 pnpm dev
```

### Q6:OAuth 启动报"私钥加载失败"

```bash
# 检查文件
ls -la keys/oauth-private-dev.pem
# 权限应该是 -rw------- (600)
chmod 600 keys/oauth-private-dev.pem
```

### Q7:测试报 "Cannot find module ../generated/prisma"

```bash
pnpm db:generate   # 重新生成 Prisma Client
```

### Q8:想完全重置

```bash
pnpm clean:db                                    # 删 PG 数据卷
rm -rf node_modules src/generated keys/*.pem     # 删依赖/生成物/密钥
pnpm setup                                       # 重新一键
```

## 8. 应用结构(快速定位)

```
src/
├── server.ts           启动入口(connect → migrate → seed → listen)
├── app.ts              Express 装配(中间件 + 路由)
├── config/             env 加载与默认值
├── database/
│   ├── prisma.ts       PrismaClient 单例(应用层 DB 唯一入口)
│   └── bigint-json.ts  BigInt → number JSON polyfill
├── middleware/         鉴权 / CSRF 中间件
├── oauth/              OAuth 2.1 / OIDC IdP 模块(见 docs/oauth/)
├── routes/             业务路由(login/register/chat/friend/user/upload)
├── utils/socket.ts     Socket.IO 通信层
└── generated/prisma/   Prisma Client(自动生成,不进 git)

prisma/
├── schema.prisma       16 个 model 的 schema 唯一来源
└── migrations/         schema 演进历史

docs/
├── oauth/              IdP 模块完整文档
├── database/           数据库重构记录 + migration SOP
└── onboarding/         本目录 ── 新人入门
```
