# 04 · 密钥管理

## 1. 密钥规格

- 算法:**RS256**(RSA-SHA256),签名长度 2048 bit
- 也可未来加 **ES256**(ECDSA-P256),验签更快、密钥更短;架构已支持多算法
- 私钥格式:**PKCS#8 PEM**
- 公钥格式:**JWK**(经 `/.well-known/jwks.json` 公开)

## 2. 密钥生成(本地 / 部署初始化)

```bash
# 1. 生成 2048 RSA 私钥(PKCS#8 PEM)
openssl genpkey -algorithm RSA -pkcs8 \
  -pkeyopt rsa_keygen_bits:2048 \
  -out keys/oauth-private-k-2026-06.pem

# 2. 抽取公钥(可选,服务启动时也能从私钥推)
openssl pkey -in keys/oauth-private-k-2026-06.pem -pubout \
  -out keys/oauth-public-k-2026-06.pem
```

文件名约定:`oauth-private-<kid>.pem`,**`kid` 直接编进文件名**,便于多 kid 共存。

## 3. 加载方式

### 3.1 单 kid(默认)

```bash
# .env
OAUTH_PRIVATE_KEY_FILE=keys/oauth-private-k-2026-06.pem
OAUTH_ACTIVE_KID=k-2026-06
```

### 3.2 多 kid(轮换期)

```bash
# .env
OAUTH_KEY_DIR=keys
OAUTH_ACTIVE_KID=k-2026-07        # 用新 kid 签发
OAUTH_RETIRED_KIDS=k-2026-06      # JWKS 仍暴露,但不再签新 token
```

启动时扫 `OAUTH_KEY_DIR/oauth-private-*.pem`,按 kid 索引,**active 用于签发,retired 仅用于公钥发布**(等已签发 token 自然过期就可彻底下线)。

### 3.3 生产部署:私钥 NOT in git

- 私钥**绝不进 git**,`.gitignore` 加 `keys/*.pem`
- 容器化:挂载 secret volume(K8s Secret / Docker secret / cloud KMS)
- 单机:经 `chmod 600` 限制读权限,只有 server 进程用户可读
- env 配置传 `KMS_KEY_ID` + 启动时拉真私钥(高级场景)

## 4. 启动校验

server 启动时执行(`src/oauth/keys.ts` 实现):

1. 加载 active kid 私钥,失败 → **fail-fast**,server 拒绝启动(因为 OAuth 模块依赖签发能力)
2. 校验 RSA modulus ≥ 2048,小于直接拒绝
3. 推算公钥并生成 JWK,塞进 JWKS endpoint 响应缓存
4. 日志输出:`OAuth keys loaded: active=<kid>, retired=[<kid>...]`,**不输出任何密钥材料**

## 5. 密钥轮换 SOP

### 5.1 轮换触发条件

- 定期轮换:每 6 个月一次(预防潜在但未被发现的泄漏)
- 紧急轮换:确认 / 怀疑泄漏 → 立即

### 5.2 标准滚动轮换流程(零停机)

```
Step 1. 生成新密钥
  openssl genpkey ... > keys/oauth-private-k-2026-12.pem

Step 2. 部署新文件,server 实例逐个滚动重启
  - 暴露双 kid:JWKS 同时返回 k-2026-06 和 k-2026-12 的公钥
  - 仍用 k-2026-06 签新 token(env 不变)
  - 所有 resource server 缓存 JWKS,自动拉到双 kid

Step 3. 等 resource server JWKS 缓存全部更新(默认 10 min)

Step 4. 切签发到新 kid
  OAUTH_ACTIVE_KID=k-2026-12       # 改 env
  OAUTH_RETIRED_KIDS=k-2026-06     # 旧 kid 仍发公钥
  滚动重启

Step 5. 等所有已用 k-2026-06 签的 RT 自然过期(最长 30 day)

Step 6. 下线旧 kid
  OAUTH_RETIRED_KIDS=             # 清空
  删 keys/oauth-private-k-2026-06.pem(归档备份后)
  滚动重启
```

### 5.3 紧急轮换(怀疑泄漏)

- 跳过 step 3 / 5 的等待时间
- 直接 `UPDATE oauth_refresh_tokens SET revoked = 1 WHERE issued_at < '<泄漏怀疑时间>'`(强制全员重登)
- 通报安全事件

## 6. 密钥备份

- 私钥**必须备份**,丢失会让所有已签发的 token 立即不可验证(但 token 自然过期后无影响)
- 备份策略:加密(用主密码)归档到与生产隔离的安全存储
- 记录"哪个 kid 在哪段时间作为 active 使用"作为审计材料

## 7. 反模式(禁止)

- ❌ 把私钥 base64 编进 env 字符串 → 日志 / 进程列表泄漏面
- ❌ 同一对密钥用于 OAuth 签发 + 其他用途(SSH / TLS)
- ❌ 用 HS256 共享 secret 假装"对称版 RS256"
- ❌ JWKS endpoint 加鉴权 → 公钥就应该是公开的,加鉴权反而暴露 issuer 可疑

## 8. JWKS 缓存策略

服务端:静态字符串,启动时构造一次,内存常驻
资源服务器侧:
- `jwks-rsa` 库默认缓存 10 min
- 推荐设 `rateLimit: true`,防爆量请求 JWKS
- 推荐设 `cacheMaxAge: 600_000`(10 min)+ 不命中时拉取 + 出错时 fallback 上次缓存
